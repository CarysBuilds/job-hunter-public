# Job Hunter 使用说明

Job Hunter 是一个在本机运行的求职助手，可以从 BOSS 直聘、猎聘和智联招聘抓取岗位，按照你的求职偏好进行匹配评分，并记录岗位和投递进度。

它不会自动投递，也不会自动给招聘者发消息。所有抓取操作都需要你手动启动。

## 下载与安装

### 普通用户

1. 在 GitHub Releases 页面下载 `JobHunter-Setup-x64.exe`。
2. 双击安装包，按照提示完成安装。
3. 从桌面快捷方式或开始菜单打开 Job Hunter。

目前安装包主要支持 Windows 10/11 x64。

Windows 可能因为安装包尚未获得商业代码签名而显示“Windows 已保护你的电脑”或其他安全提醒。这不代表程序一定有问题。请确认安装包来自本项目的 GitHub Releases 页面，然后点击“更多信息”→“仍要运行”继续安装。如果安装包来自网盘、群聊或其他不明来源，请不要直接运行。

### 熟悉开发的用户

可以直接查看源代码，并在本地安装、测试和运行：

```bash
npm install
npm run typecheck
npm test
npm run dev
```

然后打开 <http://localhost:3000>。

## 第一次使用

首次打开后，先进入“设置”填写以下内容：

- 抓取关键词：你想找的岗位名称，例如“产品经理”“Java 开发”“新媒体运营”。系统会按照每个关键词分别在招聘平台搜索。
- 目标方向：用于判断抓取到的岗位是否符合你的求职方向，只影响匹配评分，不会代替抓取关键词。
- 抓取城市：直接填写城市名称，例如“北京”。如果同时找多个城市，用逗号分隔，例如“北京,上海,杭州”，最多填写 5 个。系统会自动转换成三个平台各自使用的城市参数，转换规则见 [城市匹配说明](docs/city-codes.md)。
- 经验、期望薪资和偏好城市：用于岗位评分和排序，不会修改招聘平台上的个人资料。
- 简历内容：用于生成打招呼草稿，保存到本机的 `data/profile/resume.md`。
- 模型 API Key：只有需要生成打招呼草稿时才需要填写。不使用该功能可以留空。

简单理解：

- “抓取关键词”决定搜什么。
- “目标方向、经验、薪资和城市偏好”决定抓取结果如何评分和排序。

## 登录三个招聘平台

抓取前需要分别登录 BOSS 直聘、猎聘和智联招聘账号：

1. 点击“打开 BOSS 登录”，在打开的 Chrome 中完成登录。
2. 点击“打开猎聘登录”，在另一个 Chrome 中完成登录。
3. 点击“打开智联登录”，在第三个 Chrome 中完成登录。

三个平台会使用三个独立的 Chrome 浏览器窗口和本地登录目录。启动和抓取期间请保持对应的 Chrome 窗口打开，不要退出浏览器或关闭登录页面所在的窗口。

应用会在每次抓取前检查对应平台的登录状态。未登录或登录失效时，会提示先完成登录，不会直接开始抓取。

这些 Chrome 窗口与平时使用的 Chrome 登录环境相互独立，因此即使你平时已经在招聘网站登录，也可能仍需在 Job Hunter 打开的窗口中重新登录一次。

## 抓取岗位

完成设置和登录后，点击“抓取 BOSS”“抓取猎聘”或“抓取智联”。

为了降低访问频率、减少触发平台风控的概率，自动抓取会在页面操作之间主动等待。关键词越多、抓取页面越多，总耗时越长。长时间停留在“正在抓取”通常不是卡死，请保留应用和对应的 Chrome 窗口，等待任务完成。

三个平台需要分别点击抓取，当前不会一次同时抓取全部平台。岗位是按照设置中的关键词逐个搜索的；如果关键词过多或过于宽泛，抓取时间和结果数量都会增加。

## 遇到登录验证或平台风控

招聘平台可能因为访问频率、登录环境变化或账号状态触发短信验证、滑块验证、安全验证或临时限制。

遇到此类情况时：

1. 立即停止重复点击抓取。
2. 回到对应的 Chrome 窗口，查看是否需要人工完成验证。
3. 如果仍然无法使用，关闭本次操作并等待一段时间后再登录和抓取。
4. 不要在短时间内反复刷新、重新登录或连续启动抓取，这可能延长限制时间。

平台风控由招聘网站决定，Job Hunter 无法绕过，也不能保证每次抓取都成功。

## 岗位评分与猎头岗位

系统会根据目标方向、经验、薪资、城市偏好、岗位描述和公司信息给岗位评分。评分只用于辅助筛选，不代表岗位一定适合或一定能获得面试，请在投递前阅读完整职位描述。

识别为猎头或招聘顾问发布的岗位会统一降为 C 级，但仍会保留在列表中，方便你自行判断。

如果修改了目标方向、经验或薪资等设置，可以点击“重新评分”，无需重新抓取岗位。

## 生成打招呼草稿

使用前需要同时满足两个条件：

1. 在“设置”中上传或粘贴至少 80 个字符的简历内容。
2. 填写可用的模型 API Key、API Base 和模型名称。

打开岗位详情后点击“生成打招呼草稿”。系统只会生成文案草稿，不会自动发送给招聘者。生成时，简历内容和当前岗位描述会发送到你配置的模型 API，请根据所使用模型服务商的隐私政策自行判断是否启用。

## 常见问题

### 为什么点击抓取后没有立即出现岗位？

系统会先检查登录状态，再按关键词逐页抓取，并在操作之间等待。请观察顶部状态提示，不要连续点击抓取按钮。

### 为什么已经在 Chrome 登录，应用仍提示未登录？

Job Hunter 为三个平台分别使用独立的 Chrome 用户目录。请使用应用通过“打开 XX 登录”启动的对应浏览器窗口完成登录。

### 可以关闭三个 Chrome 窗口吗？

不抓取时可以关闭。开始抓取前需要重新打开对应平台窗口并确认登录状态；抓取过程中不要关闭。

### 关键词应该怎么填？

填写招聘网站中常见的岗位名称，并用逗号分隔。例如：

```text
产品经理,产品运营,用户运营
```

建议先使用 1～3 个准确关键词测试。关键词太宽泛会带来大量不相关岗位，关键词太具体则可能没有结果。

### 抓取城市和偏好城市有什么区别？

抓取城市用于招聘平台搜索；偏好城市用于本地评分。只填写偏好城市不会自动改变平台的搜索城市。填写多个抓取城市后，系统会按照“每个关键词 × 每个城市”依次搜索，因此城市越多，总耗时越长。平台参数的转换规则见 [城市匹配说明](docs/city-codes.md)。

### 为什么同一个岗位会出现多次？

同一职位可能由不同平台、不同发布者或不同链接发布。系统会去除内容完全相同的重复数据，但会保留来源、薪资、地点或职位描述不同的记录。

### 为什么有些岗位信息不完整？

招聘平台列表页展示的信息有限，详情页也可能需要登录、验证或已经下线。系统会尽量保留可读取的内容，不完整信息会影响评分准确度。

### API Key 是 Job Hunter 提供的吗？

不是。API Key 需要向你选择的模型服务商申请，使用费用和额度也由该服务商收取。仅抓取和查看岗位不需要 API Key。

## 本地数据与隐私

- 岗位数据库、设置、简历、日志和三个平台的 Chrome 用户目录保存在本机。
- 应用不会导出招聘平台 Cookie。
- 只有手动点击抓取后，应用才会访问对应招聘平台。
- 本地服务默认只允许本机访问。
- 生成打招呼草稿时，只会把本地简历和所选岗位描述发送到你自行配置的模型 API。
- 卸载程序不一定会自动删除全部本地数据。如需彻底清理，请同时删除 Job Hunter 的本地数据目录。

更多信息见 [隐私说明](docs/privacy.md) 和 [平台访问说明](docs/platform-policy.md)。

## 使用提醒

请合理控制抓取频率，并遵守招聘平台的用户协议和适用法律。招聘网站页面结构或登录规则发生变化后，部分功能可能暂时失效，请以项目后续版本为准。
